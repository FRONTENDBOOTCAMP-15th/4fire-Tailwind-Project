## 💬 커밋 메시지 규칙

**형식**: `[타입] 작업 내용`

- `feat`: 새로운 기능 추가  
- `fix`: 버그 수정  
- `docs`: 문서 변경  
- `style`: 스타일 수정 (세미콜론 누락, 들여쓰기 등)  
- `refactor`: 코드 리팩토링  
- `chore`: 기타 변경사항

---

## 🔧 협업 규칙

- 각자 **담당 파트 기준**으로 브랜치 분기  
- 작업 전 **이슈 생성 필수**
- PR 생성 시:
  - 간단한 작업 설명 포함
  - 관련 커밋 또는 이슈 연결 (`#이슈번호`)
  - **리뷰 최소 1명 승인 후 머지**

- **코드 스타일**:
  - `Prettier` 기반 자동 정렬 사용
  - 스타일 관련 변경은 `[style]` 커밋 타입 사용
